# Recordings-first, customer-controlled processing

Studentlytics will make uploaded recordings dependable before adding live Zoom, Google Meet, Teams, or room-stream adapters. This keeps the core attendance, presence-window, engagement, evidence, consent, and reporting model provider-independent while preserving the privacy promise that biometric processing should run in a customer-controlled environment whenever possible.

**Status**: accepted

**Considered Options**: Start with live meeting bots first; outsource recognition to a cloud biometric API; build a recordings-first processing path and add live adapters after the analysis model is stable.

**Consequences**: The MVP can be tested with real recordings immediately, but production live-call support remains a roadmap item. UI should describe live support as planned unless an actual live adapter is connected.
